Part I: Introduction

Chapter 1: Welcome to Veriflow

1.1 What is Veriflow?

Veriflow is a professional media verification and workflow management application designed for video and audio professionals working in production and post-production environments. It provides a comprehensive suite of tools for secure file copying, media playback, synchronization, transcoding, and report generation.

Core Philosophy: Veriflow is built around the principle of data integrity and professional workflow efficiency. Every feature is designed to ensure your media files are handled safely, verified accurately, and documented thoroughly.

1.2 Who Should Use Veriflow?

Veriflow is designed for:

  • DITs (Digital Imaging Technicians): Manage camera media, create camera reports, verify file integrity, and prepare deliverables
  • Sound Recordists: Handle audio files, create sound reports, manage multi-track recordings, and prepare audio deliverables
  • Post-Production Professionals: Organize media, synchronize audio/video, transcode files, and maintain project documentation
  • Data Managers: Ensure data integrity, create backups, verify checksums, and maintain comprehensive logs
  • Production Coordinators: Manage media workflows, generate reports, and coordinate between departments

1.3 Key Features Overview

Dual-Destination Secure Copy

Copy files to two destinations simultaneously with xxHash64 verification. Ensures perfect data integrity with cryptographic hash comparison. Essential for critical media backup workflows.

Professional Media Playback

Frame-accurate video playback and multi-track audio playback (up to 32 tracks) with comprehensive metadata display. Supports industry-standard formats and codecs.

Audio Waveform Synchronization

FFT-based cross-correlation analysis for precise audio synchronization. Automatically detect time offsets between audio files with sub-frame accuracy.

Batch Transcoding

Convert media files between formats with support for multiple codecs, resolutions, and quality settings. Queue-based processing for efficient batch operations.

Professional Report Generation

Create industry-standard Camera Reports and Sound Reports in PDF format. Includes all essential metadata, technical specifications, and production notes.

Dual-Profile System

Switch between Video and Audio profiles optimized for different workflows. UI and features adapt automatically to your current mode of operation.

Chapter 4: Core Concepts

4.1 Sessions Explained

Sessions are the foundation of Veriflow's project organization. A session saves your complete workspace state including:

  • All loaded media files (references, not copies)
  • Report data and metadata
  • Logged clips and EDL information
  • Sync analysis results
  • Transcode queue and settings
  • Current profile (Audio/Video)
  • Page-specific settings and states

Session File Format

Sessions are saved as .vfsession files in JSON format. These files are:

  • Human-readable: Can be opened in text editor if needed
  • Portable: Can be shared between computers (file paths may need adjustment)
  • Version-controlled: Can be tracked in Git or other VCS
  • Lightweight: Typically 10-100 KB depending on content

Session Best Practices

  • Create one session per project or shoot day
  • Use descriptive names: "ProjectName_Date_Description.vfsession"
  • Save sessions in a dedicated folder, not with media files
  • Save regularly (Ctrl+S) to preserve your work
  • Keep session files backed up separately from media

4.3 File Management Philosophy

Veriflow follows professional data management principles:

Non-Destructive Operations

Veriflow never modifies your original media files. All operations are:

  • Read-only: Original files remain untouched
  • Reference-based: Sessions store file paths, not file copies
  • Safe: No risk of accidental data loss or corruption

Hash Verification

The Secure Copy feature uses xxHash64 for cryptographic verification:

  • Source file hash is calculated before copy
  • Destination file hash is calculated after copy
  • Hashes are compared to ensure perfect copy
  • Any mismatch triggers an error alert

Metadata Handling

Veriflow reads and displays embedded metadata from:

  • Video files: Codec, resolution, frame rate, timecode, camera settings
  • Audio files: BWF metadata, iXML, sample rate, bit depth, track names
  • Container metadata: Creation date, duration, file size

5.1 Overview and Purpose

The Secure Copy page is Veriflow's professional data integrity solution for copying media files to dual destinations with cryptographic hash verification.

  • Simultaneous copy to two independent destinations

  • Real-time progress monitoring

5.2 Interface Elements

Source Section

  • OPEN Button: Opens file/folder browser
  • Source Path Display: Shows full path

Destination Sections (A and B)

  • OPEN Button: Select destination folder
  • Path Display: Shows destination path

5.3 Hash Verification

Veriflow uses xxHash64 for fast, reliable file verification:

  1. Calculate hash of source file
  2. Copy file to destination(s)
  3. Calculate hash of copied file(s)
  4. Compare hashes to ensure perfect copy

Why Hash Verification Matters

Traditional copying only checks file size. Hash verification ensures bit-perfect copies by comparing actual file contents.

5.4 Workflow

  1. Click OPEN (Source) to select your media card.
  2. Click LINK A and choose primary destination
  3. Click LINK B and choose secondary destination
  4. Verify sufficient space on both destinations
  5. Click START COPY
  6. Monitor progress and wait for completion
  7. Verify green checkmarks (✓) on both destinations

5.5 Best Practices

Critical: Use Separate Physical Drives

Destinations A and B must be on separate physical drives for true redundancy.

  • Always use dual destinations
  • Verify drive health before copying
  • Don't remove source until both copies verified
  • Monitor the entire process
  • Check final verification results

5.6 Troubleshooting

Slow Copy Speed:

  • Use USB 3.0+ ports and cables
  • Close unnecessary applications
  • Check drive health

Hash Mismatch:

  • Delete corrupted copy and retry
  • Check drive health and cables
  • Test with different USB port

Chapter 8: Sync Page

8.1 Overview and Purpose

The Sync page uses FFT-based cross-correlation analysis to automatically detect time offsets between audio files. This is essential for synchronizing separately recorded audio with video or aligning multiple audio sources.

8.2 How Audio Sync Works

FFT Cross-Correlation

Veriflow uses advanced signal processing to find time alignment:

  1. Convert to Frequency Domain: FFT analysis of both audio files
  2. Cross-Correlation: Mathematical comparison to find best alignment
  3. Peak Detection: Identify time offset with highest correlation
  4. Sub-Frame Accuracy: Precise offset calculation

8.3 Interface Elements

Video Pool

  • IMPORT VIDEO: Select the video or reference audio files
  • List View: Shows filename, start timecode, and duration
  • CLEAR LIST: Remove all items from the video pool

Audio Pool

  • IMPORT AUDIO: Select the audio files to synchronize
  • List View: Shows filename, start timecode, and duration
  • CLEAR LIST: Remove all items from the audio pool

Matched Pairs

After analysis, the matched list displays:

  • Video Master: The reference file name
  • Audio Source: The synchronized file name
  • Offset: Detected time shift (e.g., +2.34s)
  • Status: Synced or Failed
  • Sync Method: Timecode or Waveform

Batch Export

  • Export Destination: Select folder for synced files
  • EXPORT ALL SYNCED: Process and save all valid matches

8.4 Workflow

  1. Load Video Pool
    • Click IMPORT VIDEO
    • Select your video or reference master files
  2. Load Audio Pool
    • Click IMPORT AUDIO
    • Select the separate audio files to synchronize
  3. Start Analysis
    • Click SYNC BY TIMECODE or SYNC BY WAVEFORM
    • Wait for processing (progress shown per file)
    • Analysis time depends on file duration and sample rate
  4. Review Results
    • Check offset values
    • Verify confidence scores (>80% is good)
    • Note any failed analyses
  5. Export Results
    • Select an Export Destination folder
    • Click EXPORT ALL SYNCED to save new files

8.5 Understanding Results

Offset Interpretation

Offset Meaning
+2.5 seconds File starts 2.5 seconds AFTER reference
-1.3 seconds File starts 1.3 seconds BEFORE reference
0.0 seconds File is perfectly aligned with reference

Status Indicators

  • Synced: A valid offset was found.
  • Failed: No correlation found (signal too different or no overlap).

8.6 Troubleshooting Sync Issues

Sync Fails:

  • Files may not have overlapping audio content
  • Audio quality too different (e.g. camera mic vs. boom)
  • Excessive background noise

Analysis Fails:

  • Ensure files have audio content (not silent)
  • Check sample rates are compatible
  • Verify files are not corrupted
  • Try shorter file segments if very long

Best Results

For best sync results:

  • Use files with clear, distinct audio events (claps, dialogue)
  • Ensure reference and sync files overlap in time
  • Higher sample rates provide more accuracy
  • Minimize background noise differences

Chapter 15: File Formats & Codecs

15.1 Video Formats

MP4 (MPEG-4 Part 14)

  • Extension: .mp4, .m4v
  • Container Type: MPEG-4
  • Common Codecs: H.264, H.265 (HEVC)
  • Audio Codecs: AAC, MP3
  • Use Cases: Universal distribution, web delivery, mobile playback
  • Pros: Widely compatible, good compression, streaming support
  • Cons: Lossy compression, limited professional codec support

MOV (QuickTime)

  • Extension: .mov
  • Container Type: QuickTime
  • Common Codecs: ProRes, H.264, H.265, DNxHD
  • Audio Codecs: PCM, AAC
  • Use Cases: Professional editing, camera originals, mastering
  • Pros: Supports professional codecs, extensive metadata, timecode
  • Cons: Larger file sizes, less universal than MP4

MXF (Material Exchange Format)

  • Extension: .mxf
  • Container Type: SMPTE standard
  • Common Codecs: DNxHD, XDCAM, AVC-Intra
  • Use Cases: Broadcast, professional workflows, archival
  • Pros: Industry standard, extensive metadata, frame-accurate
  • Cons: Complex format, limited consumer support

AVI (Audio Video Interleave)

  • Extension: .avi
  • Container Type: Microsoft RIFF
  • Common Codecs: DV, MJPEG, various legacy codecs
  • Use Cases: Legacy systems, DV tape capture
  • Pros: Simple structure, widely supported historically
  • Cons: Outdated, limited modern codec support, large files

MKV (Matroska)

  • Extension: .mkv
  • Container Type: Matroska
  • Common Codecs: H.264, H.265, VP9, AV1
  • Use Cases: Open-source projects, archival, multi-track
  • Pros: Open format, supports unlimited tracks, extensive features
  • Cons: Limited professional tool support

15.3 Audio Formats

WAV (Waveform Audio File Format)

  • Extension: .wav

  • Sample Rates: 44.1, 48, 96, 192 kHz
  • Bit Depths: 16, 24, 32-bit (float or integer)
  • Channels: Mono, stereo, multi-channel
  • Metadata: BWF (Broadcast Wave Format) extension
  • Use Cases: Professional audio, mastering, archival

BWF (Broadcast Wave Format)

BWF is an extension of WAV that adds professional metadata:

  • Description: File description text
  • Originator: Creator/organization
  • Originator Reference: Unique identifier
  • Origination Date/Time: Creation timestamp
  • Time Reference: Sample-accurate timestamp
  • Coding History: Processing history

MP3 (MPEG-1 Audio Layer 3)

  • Extension: .mp3
  • Compression: Lossy
  • Bit Rates: 128-320 kbps (typical)
  • Use Cases: Distribution, web, mobile
  • Pros: Universal support, small files
  • Cons: Lossy, not suitable for mastering

AAC (Advanced Audio Coding)

  • Extension: .aac, .m4a
  • Compression: Lossy, better than MP3
  • Bit Rates: 128-320 kbps
  • Use Cases: Modern distribution, streaming
  • Pros: Better quality than MP3 at same bitrate

FLAC (Free Lossless Audio Codec)

  • Extension: .flac
  • Compression: Lossless (~50% size reduction)
  • Use Cases: Archival, high-quality distribution
  • Pros: Lossless, open format, good compression

19.1 Installation & Startup Issues

19.2 Secure Copy Issues

Copy Operation Freezes

Symptoms: Progress bar stops, no activity for several minutes.

Diagnosis:

  • Check if drive activity LED is still blinking
  • Open Task Manager → Performance → Check disk activity
  • Wait 2-3 minutes (large files may appear frozen)

Solutions:

  1. If truly frozen, click CANCEL and retry
  2. Try different USB port (use motherboard ports, not hub)
  3. Replace USB cable
  4. Check drive health with manufacturer tools
  5. Disable antivirus real-time scanning temporarily

Hash Verification Failures

Symptoms: Red X marks, "Hash mismatch" errors.

Causes:

  • Failing source drive
  • Failing destination drive
  • Bad USB cable or connection
  • USB power issues
  • Drive disconnection during copy

Solutions:

  1. Immediate: Delete corrupted files, retry copy
  2. Hardware:
    • Replace USB cable
    • Use different USB port
    • Use powered USB hub
    • Connect drives directly to motherboard ports
  3. Drive Health:
    • Run CHKDSK on drives
    • Check SMART status
    • Test with different drives

Critical: Repeated Hash Failures

If hash verification fails repeatedly on the same files:

  • STOP using the suspect drive immediately
  • The drive may be failing
  • Consult data recovery professional if data is critical
  • Replace drive before continuing production

Slow Copy Speed

Expected Speeds:

  • USB 2.0: 20-40 MB/s
  • USB 3.0: 100-150 MB/s
  • USB 3.1/3.2: 400-1000 MB/s

If speeds are much lower:

  1. Verify USB 3.0+ port (blue connector)
  2. Use USB 3.0+ cable
  3. Close background applications
  4. Disable antivirus scanning temporarily
  5. Check drive isn't fragmented (HDD only)
  6. Update USB drivers

19.3 Media & Player Issues

File Won't Load in Player

Symptoms: Error message when trying to load file, or player shows black screen.

Solutions:

  1. Verify file isn't corrupted (try in VLC or other player)
  2. Check file format is supported (see Chapter 15)
  3. Ensure file isn't locked by another application
  4. Try copying file to local drive first
  5. Check application logs for specific error

No Audio Playback

Symptoms: Video plays but no sound, or audio file shows waveform but silent.

Solutions:

  1. Check Windows volume mixer (Veriflow not muted)
  2. Verify correct audio output device selected in Windows
  3. Check track isn't muted in Player (Audio profile)
  4. Verify file actually contains audio (check in Media page metadata)
  5. Try different audio file to isolate issue

Choppy Video Playback

Symptoms: Video stutters, drops frames, or plays slowly.

Solutions:

  1. Close unnecessary applications
  2. Copy file to local SSD before playback
  3. Update graphics drivers
  4. Reduce video resolution (4K/8K may struggle on older hardware)
  5. Check CPU usage in Task Manager

19.4 Sync Page Issues

Low Confidence Scores

Symptoms: Sync analysis completes but confidence < 70%.

Causes:

  • Files don't overlap in time
  • Very different audio quality (camera mic vs boom)
  • Excessive background noise
  • Files from different takes/scenes

Solutions:

  1. Verify files are from same take/scene
  2. Check files have overlapping audio content
  3. Try shorter file segments
  4. Manually verify sync with Player

Sync Analysis Fails

Error Messages: "Analysis failed", "Unable to process"

Solutions:

  1. Ensure files contain actual audio (not silent)
  2. Check files aren't corrupted
  3. Verify compatible sample rates
  4. Try with shorter file duration
  5. Check application logs for details

19.5 Transcode Issues

Transcode Fails

Symptoms: File shows "Failed" status in queue.

Solutions:

  1. Check source file isn't corrupted
  2. Verify sufficient disk space on output drive
  3. Ensure output folder exists and is writable
  4. Try different codec/format combination
  5. Check logs for specific error message

Very Slow Transcode

Expected: 0.5-2x realtime depending on codec and system.

  1. Close all other applications
  2. Check CPU isn't thermal throttling
  3. H.265 is slower than H.264 (expected)

  5. Ensure adequate cooling for CPU

Appendix A: Glossary

AAC (Advanced Audio Coding)
Lossy audio compression format, successor to MP3 with better quality at same bitrate.
AVC (Advanced Video Coding)
Also known as H.264, widely used video compression standard.
Bit Depth
Number of bits used to represent each audio sample. Common values: 16-bit, 24-bit, 32-bit. Higher = better dynamic range.
Bitrate
Data rate of compressed media, measured in Mbps (megabits per second). Higher = better quality, larger files.
BWF (Broadcast Wave Format)
Professional extension of WAV format that includes extensive metadata fields.
Checksum
Mathematical value calculated from file data, used to verify file integrity. See also: Hash.
Codec
Compressor/Decompressor - algorithm used to encode and decode media. Examples: H.264, ProRes, AAC.
Container Format
File format that holds video, audio, and metadata. Examples: MP4, MOV, MXF.
DIT (Digital Imaging Technician)
On-set professional responsible for camera data management, quality control, and media backup.
DNxHD/DNxHR
Avid's professional video codecs. DNxHD for HD, DNxHR for UHD/4K and higher.
EDL (Edit Decision List)
Text file containing edit points (in/out, timecode) for use in NLE systems.
FFT (Fast Fourier Transform)
Mathematical algorithm used to convert audio from time domain to frequency domain, used in sync analysis.
Frame Rate (FPS)
Frames per second - how many images displayed per second. Common: 23.976, 24, 25, 29.97, 30, 60 fps.
Hash
Unique digital fingerprint of a file. Used to verify perfect copies. Veriflow uses xxHash64.
HEVC (High Efficiency Video Coding)
Also known as H.265, successor to H.264 with ~50% better compression.
iXML
Metadata standard for production audio, includes scene/take info, track names, notes.
Lossless
Compression that preserves all original data. Examples: ProRes, FLAC, WAV.
Lossy
Compression that discards some data to achieve smaller files. Examples: H.264, MP3, AAC.
NLE (Non-Linear Editor)
Video editing software. Examples: Premiere Pro, DaVinci Resolve, Avid Media Composer.
PCM (Pulse Code Modulation)
Uncompressed digital audio format, used in WAV files.
ProRes
Apple's professional video codec family, widely used in post-production.
Proxy
Lower-resolution copy of media file for faster editing. Typically H.264 720p or 1080p.
Sample Rate
Number of audio samples per second, measured in kHz. Common: 44.1, 48, 96, 192 kHz.
Session
Veriflow project file (.vfsession) that stores workspace state, file references, and metadata.
Timecode
Time reference embedded in media, format: HH:MM:SS:FF (hours:minutes:seconds:frames).
Transcode
Convert media from one format/codec to another.
xxHash64
Fast, non-cryptographic hash function used by Veriflow for file verification.

Appendix E: Common Error Messages

Copy & Verification Errors

"Insufficient space on destination"

Cause: Destination drive doesn't have enough free space.

Solution: Free up space or choose different destination.

"Hash verification failed"

Cause: Copied file doesn't match source (corruption during copy).

Solution: Delete corrupted file, retry copy. Check cables and drives.

"Access denied"

Cause: Insufficient permissions on source or destination.

Solution: Run as administrator or check folder permissions.

"File in use by another process"

Cause: Source file is locked by another application.

Solution: Close applications using the file.

Media & Playback Errors

"Unable to load media file"

Cause: File corrupted, unsupported format, or locked.

Solution: Verify file in other player, check format support.

"Codec not supported"

Cause: File uses unsupported codec.

Solution: Transcode to supported format (H.264, ProRes).

"No audio device found"

Cause: No audio output device available.

Solution: Check Windows audio settings, connect speakers/headphones.

Sync Errors

"Sync analysis failed"

Cause: Files incompatible, corrupted, or no overlapping audio.

Solution: Verify files are from same take, contain audio.

"Low confidence result"

Cause: Poor audio match between files.

Solution: Verify files overlap in time, manually check sync.

Transcode Errors

"Transcode failed"

Cause: Source file issue, insufficient space, or codec error.

Solution: Check source file, verify disk space, try different codec.

"Output folder not writable"

Cause: No write permission on output folder.

Solution: Choose different folder or fix permissions.

Session Errors

"Unable to open session file"

Cause: File corrupted or incompatible version.

Solution: Restore from backup or create new session.

"Media files not found"

Cause: Referenced files moved or drives disconnected.

Solution: Connect drives or relocate files.

System Errors

"This application requires .NET 8.0 Runtime"

Cause: Required runtime not installed.

Solution: Download and install .NET 8.0 Desktop Runtime from Microsoft.

"Out of memory"

Cause: Insufficient RAM for operation.

Solution: Close other applications, add more RAM, process smaller files.

If Error Not Listed

Check application logs (Help → Open Log Folder) for detailed error information. Include log details when reporting issues.
